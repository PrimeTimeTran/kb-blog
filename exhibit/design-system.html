Exhibit System — Signal Hierarchy Model

v1.3 layout theory • structural vs semantic vs emphasis separation • rendering pipeline model

This system is not about “styling content” — it is about separating *what a thing is structurally*, *what it means*, and *how it should be read*. Most UI inconsistency comes from collapsing these layers into a single visual signal.

Semantic Layer Proposition (Structure) Explanation Layer Warning Signal Invalid / Broken Resolved State

Core Model — Signal Hierarchy

UI explanations are not styled documents — they are a layered rendering system:

  • Structural signal → layout ownership (grid, scroll, containment)
  • Semantic signal → meaning classification (info, warning, fixed, wrong)
  • Emphasis signal → typography inside sentences (bold / italic / strike)
  • Decorative signal → icons, gradients, visual aids (never meaning-bearing)
The key failure mode in most systems is signal stacking:

A “warning” becomes red + bold + icon + border + background — which destroys hierarchy because multiple layers are encoding the same meaning.

Setup Surface — Typography as Structural Tool

Core Principle

Typography is often misused as emphasis. In this system, it is treated as a structural readability layer, not a semantic signal. It exists to guide scanning, not to inject meaning.

Correct Separation Rule

  • Bold → only used inside semantic statements (definition / constraint)
  • Italic → only used inside interpretive layers (mental model)
  • Strikethrough → only used to show invalidated assumptions

Implementation

/* structural typography rules */ b { font-weight: 600; } /* constraint marker */ i { font-style: italic; } /* interpretive layer */ s { opacity: 0.6; } /* invalidation marker */

Interpretation

In neutral contexts, typography should not introduce new meaning — only organize existing meaning.

Setup Surface — Iconography as Non-Semantic Decoration

Core Principle

Icons are frequently mistaken for meaning carriers. In this system, icons are strictly navigation and scan accelerators, never semantic indicators.

Allowed Usage

  • visual grouping (e.g. section identity at a glance)
  • scan enhancement (faster recognition of structure)
  • decorative reinforcement of already-defined roles

Forbidden Usage

  • using icons to define “importance” instead of roles
  • using icons to replace semantic labels (info/warn/fixed)
  • stacking icons with color to double-encode meaning

Implementation

/* icon system (neutral only) */ .icon { display: inline-flex; opacity: 0.7; /* never primary signal */ margin-right: 6px; } /* rule: icons never define role */

Interpretation

Icons should reduce cognitive load, not create additional interpretation layers.

Proposition — Structural Layer
Layout behavior is determined by containment rules, not visual styling.
Semantic Layer — Meaning Only
Color should classify intent, not amplify importance.

Example: “info” does not mean less important — it means not structural.
Wrong — Signal Collapse
Mixing structural + semantic + emphasis signals into one visual treatment causes ambiguity:

Everything looks important → nothing is prioritized.
Fixed — Separated Signal Layers
Each layer encodes one responsibility:

structure = layout
meaning = color
emphasis = typography
