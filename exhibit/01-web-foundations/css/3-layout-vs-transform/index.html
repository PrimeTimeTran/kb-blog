Layout vs Transform

A visual explanation of why flex layout and pixel offsets feel simple in CSS — but become unstable when merged inside a scene graph system.

1. The Mental Model We THINK We Have

“Flex lays things out, then offsets adjust position slightly.”

This feels correct because CSS *lets it appear unified* — but internally it is two separate systems.

3. The Separation That Makes It Stable

Layout determines structure. Transform only tweaks pixels.

The system becomes predictable when “offset” stops affecting layout.
layout: flex → x, y, width, height
transform: translate → visual adjustment only

The Core Insight

The confusion isn’t about flex or layout systems — it’s about accidentally merging layout space and visual space into one concept.

Once separated, the system stops being “fragile code” and becomes a predictable scene graph.
