The Real Problem Is Not Flex

It is boundary ambiguity between 3 systems:

  • Layout system (flex / width / height)
  • Interaction system (hover / mouseenter / leave)
  • Position system (fixed overlays)

Minimal reproduction of the failure

fixed container
  + flex layout
  + hover-driven state
  + dropdown expansion
  + second fixed layer (bottom trap)

What this causes (emergent behavior)

  • Mouse never fully exits interaction space
  • Hover state feels “sticky” or re-triggered
  • Dropdown feels like it affects page layout
  • Escape requires artificial spacing hacks

Key insight

A layout stops being “visual structure” when it becomes part of the interaction boundary.

Why “just flex” does not fix it

  • Flex only distributes space
  • It does not define interaction boundaries
  • It becomes part of the problem when used inside fixed hover systems

Correct mental model

    SCREEN = static page

    NAV = interaction layer

    DROPDOWN = temporary overlay system

    TRAPS (bottom / invisible divs) = accidental state controllers
